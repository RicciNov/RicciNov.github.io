UUID

UUID的全称，Universally Unique Identifier，通用唯一识别码。这个很清楚，凡是编程过的人一般都会立刻嘴角上扬了。毕竟在实际代码工程中，要给某个对象也好、线程也好、或者其它，使用一个唯一的、不会重复的ID，很重要，就像数据库中的主键。目前最广泛应用的UUID，就是微软的GUID(Globally Unique Identifier)。

UUID的生成规范定义的算法主要目的就是要保证其唯一性，当然这个唯一性是有限的，只有在特定的范围内才能得到保证，这和UUID的类型有关。后面细说。

UUID是16字节（1Byte = 8bits）128位长的数字，通常以36位长度的16进制数表示，示例如下：

3F2504E0-4F89-11D3-9A0C-0305E82C3301

其中字母是16进制表示，大小写无关。

具体来介绍UUID的内涵，可以从如下几个方面展开：

  • 经由一定的算法生成
    为了保证UUID的唯一性，规范定义了包括网卡Mac地址、时间戳、名字空间(Namespace)、随机或伪随机数、时序等元素，以及以这些元素生成UUID的算法。UUID的复杂特性也保证了其唯一性的同时，意味着只能由计算机生成。
  • 非人工指定，非人工识别
    UUID不能也不可能是人工指定的，除非冒着重复ID的风险。UUID的复杂性也决定了无法简单地分辨出UUID和对象之间的映射关系。
  • 在指定的范围内重复的可能性极小
    UUID的生成规范定义的算法主要目的是要保证其唯一性，但这个唯一性也是有限的，从特定的名字空间到全球。只有在特定的范围内才能真正保证其唯一性。

UUID有多个版本，每个版本的规范算法不同，应用范围也不同。
首先有一个特例，也就是Ni1 UUID，通常不会用到，其全部由0组成，如下：

00000000-0000-0000-0000-000000000000

  • Version 1：基于时间的UUID
    基于时间的UUID通过计算当前时间戳、随机数和机器Mac地址得到。由于算法中使用了Mac地址，这个版本的UUID可以保证在全球范围的唯一性；当然，也同样带来了安全问题。如果应用只是在局域网内使用，也可以考虑退化的算法，以IP地址代替Mac地址（Java的UUID往往是这样实现的，当然也是考虑了获取Mac的难度）。
  • Version 2：DCE安全的UUID
    DCE(Distributed Computing Environment)安全的UUID和基于时间的UUID算法相同，但会把时间戳的前4位置换成POSIX的UID或GID。这个版本的UUID在实际中应用较少。
  • Version 3：基于名字的UUID（MD5）
    基于名字的UUDI通过计算名字和名字空间的MD5散列值得到。这个版本的UUID保证了：相同名字空间中不同名字生成的UUID的唯一性；不同名字空间中的UUID的唯一性；相同名字空间中相同名字的UUID重复生成是相同的。
  • Version 4：随机UUID
    根据随机数，或者伪随机数生成UUID。这种UUID产生重复的概率是可以计算出来的。
  • Version 5：基于名字的UUID（SHA1）
    和版本3的UUID算法类似，只是散列值计算使用SHA1（Secure Hash Algorithm 1）算法。

从UUID的不同版本可以看出，Version 1/2适合应用于分布式计算环境下，具有高度的唯一性；Version 3/5适合于一定范围内名字唯一，且需要或可能会重复生成UUID的环境下；至于Version 4，一般不建议使用。
通常在标识对象或持久化数据时，推荐使用UUID作为唯一标识；但是以下情况不建议使用UUID：

  • 映射类型的对象，比如只有代码及名称的代码表。
  • 人工维护的非系统生成对象，比如系统中的部分基础数据。

对于具有名称不会重复的自然特性的对象，建议使用Version3/5的UUID，比如系统中的用户。如果用户的UUID使用Version 1的，那么如果在删除该用户后再次重建时，就很不会再生成同样的UUID，相当于人还是那个人，但是用户已经不是那个用户了。

 

Python中的UUID

Python在Lib/uuid.py模块中提供了针对UUID的支持。通过几个方法分别可以得到如上版本的UUID对象，不过Version 2的UUID，当前Python中暂时没有支持。其它的，uuid.uuid1()方法生成对应Version 1，uuid.uuid3()方法生成对应Version 3，uuid.uuid4()方法生成对应Version 4， uuid.uuid5()方法生成对应Version 5。

同样如上，由于uuid4()存在概率性重复，且无映射性，最好不要使用。
